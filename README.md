# Project Roadmap

## ✅ Phase 00 — Standards

Establish project standards, naming conventions, documentation, and development workflow.

## 🚧 Phase 01 — Shell Toolkit

Develop a reusable Bash toolkit including aliases, functions, automation scripts, and an integrated help system.

## 🚧 Phase 02 — Technician Repository

Create standardized hardware repositories for Dell and other manufacturers containing drivers, firmware, manuals, diagnostics, and deployment documentation.

## ⏳ Phase 03 — Rescue USB

Build a comprehensive Ventoy-based technician toolkit for diagnostics, recovery, deployment, and repair.

## ⏳ Phase 04 — Desktop Engineering

Document and standardize the Linux workstation including NVIDIA, kernel management, automation, and recovery procedures.

## ⏳ Phase 05 — Captain Cronos Linux

Build a customized Ubuntu Technician Edition with preconfigured tools, scripts, documentation, and automated deployment.

## ⏳ Phase 06 — TrueNAS

Develop a resilient storage platform supporting backups, media services, virtualization, and project archives.

## ⏳ Phase 07 — Bitcoin Infrastructure

Maintain a self-hosted Bitcoin ecosystem including a full node, Electrs, RPC Explorer, backups, and recovery documentation.

## ⏳ Phase 08 — Documentation

Build a complete technical knowledge base covering Linux, networking, storage, recovery, diesel technology, and standard operating procedures.

## ⏳ Phase 09 — Automation

Automate maintenance, reporting, monitoring, backups, validation, and system health checks across the entire environment.

# Design Philosophy

* Keep solutions simple and maintainable.
* Automate repetitive tasks whenever practical.
* Document everything worth repeating.
* Prefer standards over one-off fixes.
* Test before deploying.
* Always maintain a recovery path.
* Build systems that can be rebuilt from documentation alone.
